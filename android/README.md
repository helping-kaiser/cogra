# CoGra Android

The reference frontend. See
[docs/implementation/android.md](../docs/implementation/android.md) for the
design and [android/CLAUDE.md](CLAUDE.md) for the assistant rules.

## Modules (slice 1)

- `core:domain` — use-cases and domain types; plain Kotlin, no Android deps.
- `core:network` — Apollo client + generated operations and the encrypted
  token store; no UI.
- `feature:auth` — login + profile Compose screens and their ViewModels.
- `app` — application shell, navigation, DI entry point.

`core:ranker` and the other `feature:*` modules land with the slices that
need them.

## First-time setup

The Gradle wrapper JAR is binary and is **not** committed by the assistant.
Generate it once before the first build:

```bash
# from android/, with a system Gradle 8.11+ installed
gradle wrapper --gradle-version 8.11.1
```

Or just open `android/` in Android Studio — it offers to create the wrapper
on first sync.

Point the build at your Android SDK with `android/local.properties`
(gitignored):

```properties
sdk.dir=/path/to/Android/Sdk
```

## Build, test, run

```bash
./gradlew :app:assembleDebug          # build the app
./gradlew test                        # JVM unit tests (all modules)
./gradlew :app:installDebug           # install on a running emulator/device
```

The GraphQL client is generated by Apollo Kotlin from the repo-root
`../schema.graphql` (the checked-in backend contract) during the
`:core:network` build — never hand-edit generated code.

## Pointing the app at a backend

The debug build targets the API at `http://10.0.2.2:8080/graphql`
(`10.0.2.2` is the host loopback as seen from the Android emulator). Start
the backend with `make dev` from the repo root before the hand test.
